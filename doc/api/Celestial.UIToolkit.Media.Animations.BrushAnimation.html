Remarks

When using this animation, be aware that it is a lot heavier than other animations, since it needs to create a new Brush instance during every animation cycle. With that being said, assuming a "normal" usage, no performance decrease will be noticeable.

The animation only supports brushes of the following types:

  • SolidColorBrush
  • LinearGradientBrush
  • RadialGradientBrush. In addition, the animation can only animate from one brush to another brush of the same type. This means that you can, for instance, animate from a LinearGradientBrush to another LinearGradientBrush, but not from a SolidColorBrush to a RadialGradientBrush. Furthermore, the animation will throw an exception if un-animatable properties like enumeration values (for example, ColorInterpolationMode) of two brushes have the same value. Finally, GradientBrush objects must always have the same number of gradient stops.
